# Project: FinBTI (내 지갑 속의 금융 성향 진단 서비스)

## 1. 기획 배경 (Background)
- **Problem:** 기존 은행 앱의 자산 관리(PFM)는 "지출을 줄이세요" 같은 잔소리형 조언 위주라, 재미와 경험을 중시하는 2030 MZ세대(잘파세대)의 이탈이 발생함.
- **Insight:** 젊은 세대는 MBTI처럼 **'나 자신을 규정하고 결과를 공유하는 문화'**에 익숙함. 금융 데이터에 '게이미피케이션'을 접목하면 거부감을 낮출 수 있음.
- **Solution:** 딱딱한 금융 데이터를 **'8가지 위트 있는 페르소나(별명)'**로 치환하여 보여주고, 성향별 맞춤형 금융 솔루션을 제공하는 인터랙티브 웹 서비스.

## 2. 프로젝트 목표 (Objectives)
- **Fun Saving:** 어렵고 지루한 금융을 '게임/놀이'처럼 경험하게 하여 BNK 앱 내 체류 시간 증대.
- **Hyper-Personalization:** 단순 연령/소득이 아닌, **'성향/가치관 데이터'**에 기반한 초개인화 상품 추천.
- **Data Acquisition:** 설문 응답 데이터를 DB화하여, 향후 마이데이터와 결합할 수 있는 정성적 고객 데이터셋(Dataset) 확보.

## 3. 핵심 기능 (Key Features)

### A. 대화형 성향 진단 (Interactive Diagnosis)
- 챗봇과 대화하듯 진행되는 12~15문항의 심리 테스트.
- **3가지 핵심 분석 축 (Dimension):**
  1. **소비 축:** 절약(Saving) vs 과소비(Spending)
  2. **위험 축:** 안전지향(Safety) vs 위험감수(Risk)
  3. **관계/태도 축:** 계획/개인(Plan/Me) vs 즉흥/관계(Impulse/Social)

### B. 8가지 금융 페르소나 매칭 (Persona Mapping)
사용자의 응답 점수를 로직으로 분석하여 가장 적합한 캐릭터 부여.

| 유형 | 별명 | 한 줄 특징 |
| :--- | :--- | :--- |
| **1. 극안전형** | **🐿️ 인간 다람쥐** | "소금기 가득! 내 사전에 낭비란 없다." (쿠폰/포인트 집착) |
| **2. 극공격형** | **🦁 불나방 투자자** | "인생은 한 방! 가즈아!" (코인/급등주 선호) |
| **3. 충동형** | **💸 걸어 다니는 ATM** | "금융치료가 필요해..." (스트레스성 지출, 배달/택배) |
| **4. 관계형** | **🎅 술자리 산타클로스** | "돈은 없어도 가오는 살아야지! 내가 쏠게!" |
| **5. 계획형** | **🤖 인간 엑셀** | "내 자산 관리에 빈틈은 없다." (체리피커, 가계부) |
| **6. 방관형** | **🛌 잠자는 숲속의 예금** | "아 재테크 귀찮아.. 그냥 통장에 둘래." |
| **7. 유행형** | **👂 유행 따라 힙스터** | "요즘 이게 핫하다며? 나도 할래!" (공모주, 엔화 투자) |
| **8. 가치소비형** | **🕯️ 혼밥의 미식가** | "남 눈치 안 봐. 오직 나를 위해 쓴다." (취미, 오마카세) |

### C. 맞춤형 솔루션 처방 (Prescription)
- 진단에서 끝나는 것이 아니라, 실제 금융 행동으로 이어지도록 유도(Call to Action).
- **매칭 예시:**
  - *인간 다람쥐* → 안정적인 고금리 **[특판 예금/파킹통장]** 추천.
  - *술자리 산타* → 회비 관리의 투명성을 위한 **[모임 통장]** 추천.
  - *불나방 투자자* → 리스크 헤지를 위한 **[채권형 펀드/ETF]** 추천.

## 4. 기술적 접근 및 데이터 설계 (Technical Architecture)

### 🛠 Tech Stack
- **Backend:** Java, Spring Boot (안정적인 비즈니스 로직 처리)
- **Frontend:** React or Vue.js (인터랙티브한 UI/UX 구현)
- **Database:** MySQL (관계형 데이터 설계)

### 📊 Data Modeling (SQLD 역량 활용)
- **정규화된 DB 설계를 통해 데이터 무결성 확보**
- **주요 엔티티(Entity):**
  - `Question_Meta`: 질문 내용, 가중치(소비/위험/관계 점수), 순서.
  - `Persona_Type`: 8가지 유형별 정의, 추천 상품 ID, 텍스트 리소스.
  - `User_Result_Log`: 사용자 응답 값, 산출된 점수(X,Y,Z축), 최종 유형 매핑 이력.

### 🧠 Analysis Logic (ADSP 역량 활용)
- **Rule-Based Scoring Algorithm:**
  - 각 문항의 답변(1~4점)을 해당 카테고리(축) 점수에 합산.
  - **Threshold Mapping:** 산출된 점수 분포에 따라 8가지 유형으로 분류하는 조건문 로직 구현.
  - *Code Logic Example:*
    ```python
    if (Risk_Score > 80) and (Spending_Score > 60):
        return "불나방 투자자"
    elif (Spending_Score > 80) and (Social_Score > 70):
        return "술자리 산타클로스"
    ...
    ```

## 5. 기대 효과 (Expected Outcome)
- **Viral Marketing:** "나 인간 다람쥐 나왔어ㅋㅋ" 식의 SNS 공유 유도를 통한 자연스러운 BNK 브랜드 홍보.
- **Data-Driven Sales:** 고객 성향을 미리 파악하고 상품을 권유하므로, 무작위 마케팅 대비 높은 가입 전환율 기대.
- **Young Image:** 보수적인 지방 은행 이미지를 탈피하고 트렌디한 디지털 뱅킹 이미지 구축.
